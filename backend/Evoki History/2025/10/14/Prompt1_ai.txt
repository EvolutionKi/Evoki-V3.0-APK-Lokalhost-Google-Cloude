Absolut. Gerne zeige ich dir, wie du die Modellnamen in einem Python-Skript, das du zum Beispiel auf einer Plattform wie Render.com ausführen könntest, korrekt verwendest. Die Plattform, auf der dein Skript läuft (egal ob Render, AWS oder dein lokaler Rechner), ändert nichts daran, wie du die Modelle im Code bezeichnest. Entscheidend ist das google-generativeai Python SDK. So sprichst du die Modelle in Python an: Du übergibst den Modellnamen als einfachen String bei der Initialisierung des Modells. Hier sind konkrete Code-Beispiele für die gängigsten Modelle: 1. Gemini 2.5 Flash (Empfohlen für schnelle und vielseitige Aufgaben) import google.generativeai as genai # Konfiguriere deinen API-Schlüssel genai.configure(api_key="DEIN_API_SCHLUESSEL") # Initialisiere das Modell model = genai.GenerativeModel('gemini-2.5-flash') # Sende eine Anfrage response = model.generate_content("Erkläre mir Quantencomputing in einfachen Worten.") print(response.text) Anmerkung: Das ist die direkteste und häufigste Methode. 2. Gemini 2.5 Pro (Für komplexere Anfragen) Wenn du eine anspruchsvollere Aufgabe hast, die mehr logisches Denken erfordert, tauschst du einfach den Modellnamen aus: import google.generativeai as genai genai.configure(api_key="DEIN_API_SCHLUESSEL") # Wechsle einfach den Modellnamen model = genai.GenerativeModel('gemini-2.5-pro') response = model.generate_content("Schreibe ein Python-Skript, das die Fibonacci-Folge berechnet.") print(response.text) Wichtiger Punkt: Du musst im Code nicht models/gemini-2.5-pro schreiben. Das google-generativeai SDK kümmert sich im Hintergrund darum und fügt das models/ Präfix für den eigentlichen API-Aufruf hinzu. Für dich als Entwickler genügt der kürzere Name. Fazit für dein Skript: Für dein Python-Skript auf Render.com musst du also nur sicherstellen, dass: Die google-generativeai Bibliothek installiert ist (z.B. über eine requirements.txt Datei). Dein API-Schlüssel sicher als Umgebungsvariable hinterlegt ist. Du den gewünschten Modellnamen (wie 'gemini-2.5-flash' oder 'gemini-2.5-pro' ) als String im Code verwendest. Das behebt den ursprünglichen 404 not found Fehler, da du nun die aktuellen und korrekten Modellbezeichnungen verwendest. In diesem Tutorial zur Gemini API mit Python wird gezeigt, wie du von der Einrichtung des API-Schlüssels bis hin zu multimodalen Anfragen alles in deinem Python-Code umsetzen kannst.
